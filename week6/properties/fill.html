CSS Keyframe Animations

Fill

The animation-fill-mode property specifies a style for the element when the animation is not playing (before it starts, after it ends, or both). CSS animations do not affect the element before the first keyframe is played or after the last keyframe is played. The animation-fill-mode property can override this behavior.It can be used to make the animation remain in its final state or start from its initial state, depending on the value you choose. There are four values for this property:

none: The default value. The animation will not apply any styles to the element before or after it is executing.

forwards: The element will retain the style values that is set by the last keyframe encountered during execution.

backwards: The element will get the style values that is set by the first keyframe (depends on animation-direction), and retain this during the animation-delay period.

both: The animation will follow the rules for both forwards and backwards, extending the animation properties in both directions.

Example


 div {
  width: 100px;
  height: 100px;
 background: red;
 position: relative;
 animation: mymove 3s;
 animation-delay: 2s;
 animation-fill-mode: both;
 animation-duration: 4s;
 animation-delay: 2s;
 animation-fill-mode: value;
 }


 @keyframes mymove {
 from {top: 0px; background-color: yellow;}
 to {top: 200px; background-color: blue;}
  }

In this example boxes will move from one up and down and changing their color of background
